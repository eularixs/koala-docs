## Install

<Tabs items={["Homebrew", "Direct download"]}>
<Tab value="Homebrew">

```bash
brew install --cask eularixs/koala/koala
```

First-time install adds the tap automatically. Updates come via `brew upgrade`.

</Tab>
<Tab value="Direct download">

Download the latest `.dmg` from [GitHub releases](https://github.com/eularixs/koala/releases/latest), open it, and drag **Koala.app** to `/Applications`.

Because the build is ad-hoc signed (open-source, no paid Developer ID), the first launch may show a Gatekeeper warning. Right-click ‚Üí **Open** ‚Üí **Open** to bypass.

</Tab>
</Tabs>

**System requirements:** macOS 14 Sonoma or later. Apple Silicon recommended; Intel works.

## Your first request

<Steps>

<Step>

### Open a new request tab

Press `‚åòN` or click **+** in the top toolbar.

</Step>

<Step>

### Type a URL

Try `https://api.github.com/zen`. Method defaults to GET ‚Äî change with the dropdown or shortcut.

</Step>

<Step>

### Send it

Press `‚åò‚Üµ` (Cmd+Enter) ‚Äî response body, headers, status, timing, and size appear on the right.

</Step>

<Step>

### Save it (optional)

Press `‚åòS` to save the request to a collection. Cmd+Click in the sidebar to make a new collection on the fly.

</Step>

</Steps>

## Create your first environment

Environments hold reusable variables (`{{baseUrl}}`, `{{apiKey}}`, ...).

<Steps>

<Step>
Sidebar ‚Üí **Environments** tab ‚Üí click **+**.
</Step>

<Step>
Name it `Development`, pick a color, click **Create**.
</Step>

<Step>
The right panel opens ‚Äî add a variable: key `baseUrl`, value `https://api.staging.example.com`.
</Step>

<Step>
Top toolbar ‚Üí click the env badge ‚Üí select `Development`. Now any request URL using `{{baseUrl}}` will resolve to your staging server.
</Step>

</Steps>

<Callout type="tip">
  Use the **Vault** (`‚åò‚áßV`) instead of regular variables for secrets ‚Äî they're
  stored in the macOS Keychain and never sync to disk in plain text.
</Callout>
