Koala ships a Hasura-style GraphQL builder. No external IDE needed.

## Quick start

<Steps>

<Step>
### New GraphQL tab

`⌘N` → method dropdown → **GraphQL**. Address bar holds your endpoint (e.g. `https://api.github.com/graphql`).
</Step>

<Step>
### Add auth

Headers tab → `Authorization: Bearer {{vault.github_token}}`. GitHub's GraphQL API requires a personal access token with `read:user` minimum.
</Step>

<Step>
### Introspect schema

Click **Refresh Schema** (⟳ icon). Koala sends a standard `IntrospectionQuery` and caches the result. Sidebar populates with types, queries, mutations, subscriptions.
</Step>

<Step>
### Compose visually

Tick fields in the schema explorer. Koala builds the query for you. Or type freehand — autocomplete on `.`, on field selection, on argument names.
</Step>

<Step>
### Send

`⌘↵`. Response panel shows the JSON data + errors array.
</Step>

</Steps>

## Variables

Variables tab — JSON object passed alongside the query:

```json
{
  "userId": "123",
  "limit": 10
}
```

Reference them in the query:

```graphql
query GetUser($userId: ID!, $limit: Int!) {
  user(id: $userId) {
    posts(first: $limit) {
      title
    }
  }
}
```

Variables themselves can reference env: `"userId": "{{currentUserId}}"`.

## Fragments

Reusable selection sets. Define once, use many times:

```graphql
fragment UserCore on User {
  id
  email
  displayName
  avatarUrl
}

query Me {
  me {
    ...UserCore
  }
}

query GetUser($id: ID!) {
  user(id: $id) {
    ...UserCore
  }
}
```

Koala recognizes `fragment` definitions and auto-suggests them in selection sets.

## Subscriptions (WebSocket)

GraphQL subscriptions over `graphql-ws` or `graphql-transport-ws`:

1. Method dropdown → **GraphQL Subscription**.
2. Endpoint: `wss://api.example.com/graphql`.
3. Auth via headers (sent in `connection_init` payload).
4. Click **Connect** — Koala opens the WS, sends `connection_init`, then `subscribe` for your query.
5. Response panel shows live frames as they arrive. Click any to inspect.

## Schema explorer features

- **Type docs panel** — click any type/field to see its description, args, return type.
- **Filter** — search box at top of explorer (`⌘F` while focused). Match on type name or description.
- **Deprecated indicator** — strikethrough + warning icon, hover for replacement guidance.
- **Schema export** — Right-click any type → "Export as SDL" → file pasted to clipboard.

## Tips

<Callout type="tip">
  Use the **Tests** tab to assert on GraphQL responses. Postman-compatible
  `koala.test()` API; see [Scripts & Tests](/docs/features/scripts-tests).
</Callout>

<Callout type="info">
  GraphQL bodies use the same Monaco editor as JSON. `⌘⇧F` pretty-prints.
  `⌘/` toggles single-line comment (works in `#` comments inside the query).
</Callout>
